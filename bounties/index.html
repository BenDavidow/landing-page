Safe Solidity Solium Rules

50 REP per rule added to Solium.


Prohibit loops without fixed bounds
Prohibit unreachable code
Prohibit use of continue statement
Prohibit use of selfdestruct/suicide
Prohibit use of assembly
Prohibit use of tx.origin
Prohibit use of integer types smaller than 256 bits
Prohibit bitshifts and bit operators
Prohibit fixedpoint (fixed and ufixed) types
Prohibit modification of for loop iteration counting variables in the loop body
Prohibit use of var / all variable types must be explicitly stated
Prohibit use of named parameters in function calls
Prohibit use of inheritance
Prohibit function overriding
Prohibit abstract functions
Functions must have an explicit return statement
An else clause must be included after else if
Only one break statement allowed per loop
Prohibit use of send
*Restrict functions to a user-specified number of lines of code*
*Functions must have a single return at the end of the function*

The debugger should have the following features:

Should work on any OS
Should work with TestRPC


Desired workflow: set breakpoint in Solidity code, run tests (e.g., using Mocha or Pyethereum tester), drop into debugging environment.
